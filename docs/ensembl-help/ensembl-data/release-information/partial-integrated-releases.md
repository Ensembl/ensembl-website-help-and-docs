# Partial and Integrated data releases


There are two types of data release available via this site: Integrated data releases and Partial data releases.


## Integrated data releases
Integrated releases provide interoperable datasets for previously released genomes across multiple species.

Data in an integrated release are fixed to the time point at which they occur. These releases are scheduled every three to four months.

Integrated releases offer long-term stability and are better suited for processes that require consistent data over time.

### Integrated data releases are linkage releases and include:
- An updated collection of all data across all species
- The most comprehensive Ensembl analysis available at the time of release
- Data that will not change over time
- Data that was made available in partial releases, but further developments are coming for whole release processes.


## Partial data releases

Partial releases provide access to new or changed data as soon as possible. They are typically scheduled every two to three weeks.

Data included in a partial release may be updated or overwritten to apply fixes or patches as needed. Hence, these releases are stable in the short term, but caution should be taken in using these data for long-term processes

## Partial data releases include:
- Assemblies
- Annotations
- Genes, Exons, Transcripts and other Features
- External References (Xrefs)
- Statistics
- Variant data
- Regulation data
- Different datasets for the same genome, which are released in different partial releases
- Data that are delivered as soon as possible



### Key features of partial release datasets:

- Partial release data is available on both Ensembl Beta and the Ensembl FTP site.

- Data released in a Partial release is subjected to automated quality control procedures however manual curation is unlikely to have taken place.

- When required, partial release data may be supplanted when updated and  access to the previous iteration of a partial release will be lost. We expect supplanting to be an infrequent event.

- A selection of partial releases will be selected to be included in an integrated release. Not all partial data releases are guaranteed to be in an integrated release.

- Any erroneous data will not be removed from previous integrated releases, but will not be added to future integrated releases. In these cases, new partial data may go forward into the next integrated release.
